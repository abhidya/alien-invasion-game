# GalagAI / alien-invasion-game

A Galaga-style alien shooter that now has two runnable surfaces:

- a dependency-free static browser demo for portfolio/GitHub Pages review;
- the original Python/Pygame implementation with legacy DQN training code.

The browser demo is intentionally not a fake score table anymore. It is a playable
canvas game that loads the checked-in ship/alien sprites and can toggle a small
trained pilot model generated by `tools/train_static_pilot.py`.

<img src="assets/demo.gif" width="1000" alt="Original Pygame Alien Invasion demo">

## Static Browser Demo

Open `index.html` from a static server:

```bash
python3 -m http.server 4173
open http://127.0.0.1:4173/
```

The repo also includes `.github/workflows/pages.yml` to deploy the static root
site to GitHub Pages from `master`.

Controls:

- Arrow keys or A/D: move
- Space: fire
- P: toggle the pilot
- Touch: drag to move, tap to fire

The pilot button uses `js/galagai-model.json` when available. If that file is
missing or fails to load, the browser falls back to a deterministic heuristic.

### Retrain the Static Pilot

This is the fast, reproducible model path for the portfolio demo.

```bash
python3 -m venv .venv
. .venv/bin/activate
python -m pip install -r requirements-static-pilot.txt
python tools/train_static_pilot.py
```

Latest checked-in training run:

```json
{
  "model": "js/galagai-model.json",
  "trainAccuracy": 0.9243,
  "evalAccuracy": 0.9277,
  "samples": 12000,
  "evalSamples": 3000
}
```

This model is a small linear softmax pilot trained against a synthetic arcade
expert policy. It is not the old Keras DQN and does not claim to solve Atari.

First create your fork of the repository, then clone using:
```
cd path/to/folder
git clone https://github.com/username/alien-invasion-game.git
```

To run the game, ensure that you have python3 and pygame library installed in 
your system.

## Python/Pygame Local Play

```bash
python3 -m venv .venv
. .venv/bin/activate
python -m pip install -r requirements.txt
cd alien_invasion
python game.py
```

The game opens a Pygame window, so run it on a machine with a desktop display available.

## Legacy DQN Training Path

The original `alien_invasion/DQN.py` and `alien_invasion/game.py` path has been
modernized for TensorFlow/Keras imports and explicit CLI options. It still needs
a full desktop ML environment because it drives Pygame frames while training.

Use Python 3.12 for the ML environment; TensorFlow is not available in every
newer Python runtime.

```bash
python3.12 -m venv .venv-ml
. .venv-ml/bin/activate
python -m pip install -r requirements-ml.txt
python -m alien_invasion.game --episodes 10 --fps 240 --weights alien_invasion/weights.weights.h5 --no-plot
```

The checked-in `alien_invasion/weights.hdf5` file is a legacy artifact. New
training writes `alien_invasion/weights.weights.h5`.

## Browser Scoreboard Demo

`server.py` can serve the static demo and exposes a `/get_scores` endpoint for score display experiments:

```bash
python -m pip install flask pymongo
python server.py
# then open http://127.0.0.1:3000/
```

The scoreboard is optional and uses local MongoDB by default. Override it with:

```bash
export GALAGAI_MONGO_URI='mongodb://127.0.0.1:27017'
export GALAGAI_MONGO_DB='highscore_db'
export GALAGAI_MONGO_COLLECTION='scores'
```

If MongoDB is unavailable, `/get_scores` returns an empty score list with a `503`
instead of crashing the Flask process.

## Demo Limitations

- Pygame play requires a graphical desktop session and keyboard input.
- The static browser pilot is trained and runnable, but it is a lightweight
  model for the portfolio demo, not the full DQN.
- The included `weights.hdf5`/DQN files are legacy experimental assets, not required for the basic manual game.
- The Flask scoreboard is optional and needs MongoDB connectivity before `/get_scores` can return real data.
